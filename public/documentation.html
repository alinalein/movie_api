Documention for the Movie_API


The API should retrieve the URL from the client , the request of the client and return a corresponding response by the server. The server will fetch the data from a database.



Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/[movie] GET
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/[genre] GET
Return data about a director (bio, birth year, death year) by name /movies/[director] GET
Allow new users to register users/register/[name] POST A JSON object about the user will be added, including the following data: {....}
Allow users to update their user info (username) users/update/[ID]/[username] PUT A text message that will tell the user that his data was updated
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) movies/add/[movie]/[ID] PUT A text message that will tell the user if the movie was added to the list
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) movies/remove/[movie]/[ID] DELETE A text message that informs the user if the movie was successfully removed
Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) /users/deregister/[ID] DELETE A text message that informs the user if the profile was successfully deleted
